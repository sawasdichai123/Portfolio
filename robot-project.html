Offline Bilingual Voice-Controlled Robot Arm

A robot arm that takes spoken Thai/English commands and uses a local LLM to decide whether to execute, ask for clarification, or reject each one — grounded by vision, running fully offline.

KAIT (Kanagawa Institute of Technology), Japan · June–July 2026 · Advisor: Prof. KAWARAZAKI Noriyuki · Target: IEEE RO-MAN / SII / RA-L

Live Research Overview → View Repository

Overview

During my research internship at KAIT, I built an offline robot-arm system that takes spoken commands in Thai or English and decides how to act. Speech is transcribed with faster-whisper, a local LLM (Qwen2.5 7B on Ollama) classifies each command as EXECUTE, ASK (clarification), or REJECT, and a YOLOv8s + HSV vision pipeline grounds the decision in the objects actually on the table. The chosen action drives an OpenMANIPULATOR-X arm in RViz2 simulation over ROS 2 — with no internet connection required. The same decision pipeline was later wired up to a self-built physical 5-DOF arm (ESP32 + PCA9685, custom inverse kinematics), which picked up real bottles from spoken commands end-to-end.

Key Findings

  • Clarification is necessary (H1). Removing the clarification step drops decision accuracy to 63.3%, and on ambiguous commands the system's "first-match" guess is systematically biased toward whichever object vision detects most confidently (14.3% vs 78.6% task success depending on scene layout, averaging 46.4%) — not a random error.
  • A local ≤7B model matches cloud-tier accuracy (H2). The local Qwen2.5 7B and a cloud model (Claude Sonnet 5) both reach 100% on the same 30 cases; the local system trades ~7 s of latency for fully offline operation and zero per-token cost.
  • ~7B is the capability threshold. Across the Qwen2.5 size ladder (0.5B → 7B), every sub-7B model collapses to EXECUTE 0% (it over-asks even fully-specified commands); all three decision states only arrive reliably at 7B — a threshold, not smooth scaling.
  • Beyond simulation — a real robot picks things up. I hand-built a 5-DOF arm (ESP32 + PCA9685, DS3218 servos, custom geometric IK) and wired the same voice → LLM → vision decision pipeline into it over serial. With an Intel RealSense for metric depth, it completed full voice-commanded pick-and-place — standby → pick → place → standby — repeatably. This is a qualitative hardware demonstration alongside the RViz2-simulated results above, not a replacement for them.
